# VeilBrowser 系统端口规划（2025 年）

## 📋 概述

本文档定义了 VeilBrowser 所有端口的使用范围和分配规则，确保系统内各服务端口不冲突，并支持大规模并发。

**核心原则**：
- ✅ 所有端口仅绑定 `127.0.0.1`（localhost only）
- ✅ 使用高位端口（ephemeral ports，49152+）避免与系统服务冲突
- ✅ 主进程统一管理端口分配
- ✅ 动态端口池自动分配和回收
- ✅ 端口范围物理隔离，避免重叠

## 📊 详细说明

### 1. CDP 调试端口池（10000-11000）
- **用途**：子进程浏览器调试。
- **管理**：由 `infra/port-pool` 统一分配。

### 2. 内部服务端口池（11001-11990）
- **用途**：Profile 预览页面 (ProfilePanel) 及其他主子进程辅助服务。
- **改进**：取消 49152+ 的随机分配，全面纳入受控管理。

### 3. 外部 API 端口（11991-12000）
- **用途**：REST API 入口。
- **策略**：仅保留 10 个端口，确保入口高度固定。

### 4. TLS 代理端口池（50000-55000）
- **用途**：代理网络层的本地隧道走廊。
- **容量**：5000 端口，支持重载并发。

## 🔒 端口分配规则

### 规则 1：物理隔离
- ✅ 不同服务类型的端口范围必须完全隔离
- ❌ 禁止端口范围重叠
- ✅ 预留足够的扩展空间

### 规则 2：动态分配
- ✅ 所有 Profile 相关端口必须动态分配
- ✅ 使用端口池管理，避免手动指定
- ✅ 端口冲突自动检测和重试

### 规则 3：自动回收
- ✅ Profile 关闭时自动释放端口
- ✅ 应用退出时清理所有端口
- ✅ 端口泄漏检测和告警

### 规则 4：本地绑定
- ✅ 所有端口仅绑定 `127.0.0.1`
- ❌ 禁止绑定 `0.0.0.0` 或公网 IP
- ✅ 确保安全性

### 规则 5：高位端口
- ✅ 使用 ephemeral ports（49152+）
- ✅ 避免与系统服务冲突
- ✅ 符合 IANA 端口分配规范

## 📝 代码规范

### 新增端口使用

**必须遵守**：
1. 在 `src/shared/constants/ports.ts` 中定义端口范围
2. 使用端口池管理（如需要）
3. 在本文档中更新端口分配表
4. 在 `.cursorrules` 中记录端口使用规则

**示例**：
```typescript
// 1. 在 ports.ts 中定义
export const PORT_RANGES = {
  NEW_SERVICE: { min: 56001, max: 57000 },
} as const;

// 2. 使用端口池（如需要）
class NewServicePortPool {
  // 实现端口池逻辑
}

// 3. 更新文档和规则
```

## 🚨 注意事项

### 端口冲突检测
- 所有端口分配前必须检测可用性
- 端口被占用时自动重试或报错
- 记录端口冲突日志

### 端口泄漏预防
- Profile 关闭时必须释放端口
- 应用退出时清理所有端口池
- 定期检查端口使用情况

### 性能考虑
- 端口分配延迟 <100ms
- 端口池初始化延迟 <1s
- 支持大规模并发（100+ Profile）
